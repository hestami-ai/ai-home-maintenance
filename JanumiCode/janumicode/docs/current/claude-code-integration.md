# Claude Code and External Execution

A key architectural feature of the JanumiCode extension is its ability to integrate with external, powerful, command-line-based AI code generation tools. JanumiCode acts as a sophisticated "governance wrapper" around these tools, ensuring that their power is used in a safe, controlled, and auditable manner.

"Claude Code" is one such external tool that is supported out of the box.

## The `EXECUTE` Phase

The final step of the JanumiCode workflow is the `EXECUTE` phase. After a proposal has been generated by the `Executor`, its assumptions have been verified by the `Verifier`, its historical context has been checked by the `Historian-Interpreter`, and the final plan has been approved by a human, the `Orchestrator` hands off the task to an external execution engine.

This engine is responsible for taking the verified plan and applying it to the user's workspace (e.g., creating or modifying files).

## The `ExecutionCLIProvider` Abstraction

To support multiple different backend tools (for "cost competitiveness" and flexibility, as noted in the source code), JanumiCode uses a generic abstraction layer called the `ExecutionCLIProvider`.

This is a TypeScript interface that defines a standard contract for any CLI-based execution tool. To integrate a new tool, a developer simply needs to create a class that implements this interface.

The required functions are:

*   **`detect()`**: Checks if the CLI tool is installed, compatible, and properly configured (e.g., with an API key).
*   **`execute()`**: Executes a command with the given proposal and context.
*   **`executeStreaming()`**: Executes a command and streams the `stdout` and `stderr` in real-time, which is used to provide live progress updates in the "Governed Stream" UI.
*   **`getCommandPreview()`**: Returns the command string that will be executed, which is used for logging and observability.

## The Claude Code CLI Provider

The integration with the "Claude Code" tool is implemented in the `ClaudeCodeCLIProvider` class. This class implements the `ExecutionCLIProvider` interface and acts as a bridge between the JanumiCode orchestrator and the `claude-code` command-line tool.

It handles:
*   Detecting if the `claude-code` CLI is available on the user's system.
*   Translating the JanumiCode workflow context into the specific arguments required by the `claude-code` CLI.
*   Invoking the CLI and capturing its output.
*   Handling any errors that the CLI might produce.

## A Governance Wrapper

This architecture reveals the core value proposition of JanumiCode: it is not just another code generation tool, but a **governance and orchestration layer** for other powerful tools.

The workflow of `INTAKE` -> `PROPOSE` -> `VERIFY` -> `HISTORICAL_CHECK` -> `REVIEW` is a rigorous, structured, and auditable process designed to ensure that the final code generation step is well-planned, fact-checked, and human-approved. JanumiCode provides the safety and predictability; the external CLI providers provide the raw power.
